# 工作流: URL 投喂

用户提供包含目标人物相关内容的网页链接时,执行本流程。

## 步骤 1: 确认目标

```
这个 URL 你希望我重点关注谁的内容?
当前已有画像: L001, L002 (列出所有 memory-*.md)
或者要创建新画像?
```

## 步骤 2: 抓取网页

调用宿主提供的网页抓取工具(不同宿主名字不同):

- Claude.ai/Claude Code: `web_fetch`
- LobsterAI: 内置浏览器能力
- Hermes: 网络工具
- OpenClaw: Playwright 子工具

如果宿主无网页抓取能力,降级:

```
我所在的环境无法直接抓取网页。请你打开链接,把含有 {npc_id} 的部分
复制粘贴给我,我按文本投喂流程处理。
```

## 步骤 3: 调用脚本做相关性筛选

```bash
python scripts/extract_segments.py \
  --content-file /tmp/fetched.txt \
  --person-alias "{用户给出的别名,如'王总'}" \
  --aliases "{其他可能称呼,逗号分隔}" \
  --npc-id "{npc_id}" \
  --output /tmp/segments.json
```

脚本输出 JSON,包含:
- 直接引述段落
- 转述段落
- 第三方评价段落
- 每段的置信度

## 步骤 4: 人工审核每个片段

向用户展示所有提取到的片段,逐条让用户确认或剔除:

```
我从这个链接里抽到 5 个可能和 {npc_id} 相关的片段:

[1/5] 📍 第 3 段 (置信度 92%)
  类型: direct (直接引述)
  
  上文: "在被问及 Q4 增长策略时"
  原文: "我们不会为了增长而牺牲毛利率,这是底线。"
  下文: "他强调,公司将坚持..."
  
  ✅ 收录 / ❌ 剔除 / ✏️ 修改归属

[2/5] 📍 第 8 段 (置信度 65%)
  类型: paraphrased (转述)
  ...
```

**自动建议剔除**:
- 置信度 < 50%
- 完全是第三方评价(evaluated 类)且与画像维度无关
- 重复内容(已有 corpus 中包含相同表述)

## 步骤 5: 来源元数据

不为每个 URL 单独建文件——直接在 corpus 的 source 字段记录:

```yaml
source:
  url: <原始URL>
  title: <网页标题>
  publish_date: <发布日期>
  fetched_at: <抓取时间>
```

这样以后看到某条 corpus,能直接知道出处。

## 步骤 6: 入库

确认收录的片段,逐条按"文本投喂"流程的步骤 4-8 处理。

## 特殊场景

### 场景 A: 视频/播客转写稿

```
我看出来这是 [YouTube/B站/播客] 链接。
- 如果页面带文字转写,我直接拉取
- 否则需要你用其他工具转写后粘贴文本给我
```

转写文本通常是连续独白,attribution 几乎都是 direct。
**注意区分主持人和嘉宾发言**,只取目标人物的部分。

### 场景 B: 内部知识库链接(飞书/Notion/Confluence)

这类需要登录,宿主的 fetch 可能拿不到:

```
这个链接需要登录,我大概率拉不到完整内容。建议:
1. 你打开链接,复制 {npc_id} 相关部分粘贴给我
2. 或者导出 PDF/Markdown 上传给我

你方便哪种?
```

### 场景 C: 微信文章/小红书帖子

注意:
- 作者自述 → direct,高价值
- 评论区 → 大多无关,需要人工筛选
- 转载内容 → quoted,要追溯原始来源

脚本应识别正文与评论的边界。

## 反例: 不要这样做

❌ 不要因为网页里出现了名字就把整段判定为 X 的发言
❌ 不要在用户审核前就把抓取内容直接入库
❌ 不要丢失原始 URL,所有从网页入库的语料必须可溯源
❌ 不要对个人社交媒体内容(私人账号、家庭照片相关)进行抓取
❌ 不要绕过用户进行任何"自动搜索 X 的更多公开信息"的尝试
